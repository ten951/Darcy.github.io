发表于 2020-06-05 | 分类于 NETWORK

network

试图去解释清楚客户端请求服务端涉及到关键问题

浏览器

生成HTTP请求

打开chrome浏览器, 输入网址 https://www.google.com/?hl=zh-cn https是协议 当然还有其他的比如 http/ftp/file/mailto等. 协议顾名思义就是数据传输的格式.

  1. 浏览器解析URL. http://web服务器名/目录名/文件名 例如: http://www.lab.glasscom.com/dir1/file1.html 目的是知道访问的目标是哪里.
  2. http协议. 定义了客户端和服务器之间交互消息内容和步骤. 其基本思路是. 客户端会向服务器发送请求消息. 请求消息中包含的内容是”对什么”和进行怎样的操作. “对什么”就是资源地址. “进行怎样的操作”就是方法. 例如GET POST等. 收到客户端的请求后. 服务器端会对内容进行解析. 依据url和方法扥这些要求来完成自己工作, 然后就结果放在响应消息中. 其中状态码表示服务器执行的结果.
  3. 对URL进行解析之后,确定了web服务器和文件名. 要依据这些信息生成HTTP消息. 这是有严格的格式规定的. 请求 {请求行(GET URL HTTP/1.1), 请求头(Content-Type: text/plain;charset=UTF-8), 消息体(传输的数据,编码依据header头中的设置).} 响应 {状态行(HTTP/1.1 200 OK) 响应头(Content-Type: application/json; charset=utf-8) 响应正文(传输的数据)}
  4. 生成HTTP消息之后, 需要委托给操作系统将消息发送给WEB服务器. 但此时的消息不具备发送到网络的能力. 想要将消息发送到网络中. 需要TCP/IP协议支持. 所以我们需要知道web服务器对应的IP地址. 这就用到了DNS服务.

向DNS服务器查询WEB服务器的IP地址

为什么要ip地址和服务器域名并用. 统一使用ip地址或者域名不就可以了么?

  1. 统一使用ip是可以的. 但是出于商业目的. 人类记住字符的能力要高于数字. 比如www.google.com很容易记住.但是背后的ip地址就很难,和电话号码一样. 出于传播
  2. 处于效率考虑网络传输时使用ip定位路由器. 因为ip的长度时32bit. 也就是4字节(Byte) 而且域名最短也要十几个字节. 出于效率

DNS查WEB域名IP地址的步骤: 如 http://www.google.com

  1. 查询机器本身的HOSTS
  2. 查询最近的DNS服务器
  3. 查询根域名.
  4. 查询 .com
  5. 查询 .google.com

直到了WEB域名对应的IP地址后. 需要委托协议栈发送消息

委托协议栈发送消息

这里的协议栈指的就是TCP/IP的四层模型(应用层,传输层,网络层,网络接口层). 应用层指的就是http file这些. 可以简单地理解为Data. 这些数据按相应的协议格式封装好数据. 例如HTTP 格式. 将Data 准备好后需要将这些数据发送到目标服务器. 就需要用到传输层协议TCP网络层协议IP等协议参与. 而这些协议非常复杂. 操作系统屏蔽这些复杂性. 例如如何和网卡交互等. 提供了程序组件Socket. 所以委托协议栈发送消息. 可以理解为按照一定顺序调用socket api的过程. 其中寻址(DNS查IP也是其中的功能)
