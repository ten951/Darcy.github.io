sed和gawk

sed编辑器

流编辑器. 可以用键盘命令来交互式的插入 删除 替换数据的文本.会在编辑器处理数据之前基于预先提供的一组规则来编辑数据流.

  1. 一次从输入汇总读取一行数据. 2. 依据所提供的编辑器命令匹配数据. 3. 按照命令修改流中的数据. 4. 将新的数据输入到stdout.

sed并不会修改文本文件的数据

替换标记

s/pattern/replacement/flags -n 表示禁止sed编辑器输出. p会替换标记会输出修改过的行.

* flags 数字 表示替换第几处模式匹配的地方
* flags g 新文本会替换所有匹配的文本
* flags p 原先行的内容要打印出来
* flags w file 将替换的结果写到文件中

替换字符

sed 's!/bin/bash!/bin/csh!' /etc/passwd 将/bin/bash替换为/bin/csh

使用地址

默认情况下命令会作用于文本数据的所有行. 提供了行寻址.指定范围

数字方式的行寻址 sed '2s/dog/cat/' data1.txt' 将修改指定的第二行文本 sed ‘2,3s/dog/cat/‘ data1.txt’ 修改指定的第二行到第三行的文本 `` sed ‘2,$s/dog/cat/‘ data1.txt’ 修改指定的第二行之后的所有的文本

文本模式过滤器 sed ‘/Samantha/s/bash/csh’ /etc/passwd 只修改用户Samantha的

使用sed处理文件

sed '1,2w text.txt' data.txt 从data.txt文件中读取前两行写入text.txt文件
sed '2r text.txt' data.txt 将text.txt内容插入data.txt第二行后

sed进阶

多行命令

单行命令是指sed编辑器读取数据流时. 他会基于换行符的位置将数据分割成行. sed编辑器根据定义好的脚本命令一次处理一行. 然后移到下一行重复这个过程.

多行命令 有时需要跨多行的数据还行特定的操作.

next命令

  1. 单行的next命令 小写n命令告诉sed编辑器移动到数据流中的下一文本行. 而不用重新回到命令的最开始再执行一遍. 记住，通常sed编辑器在移动到数据流中的下一文本行之前，会在当前行上执行完所有定义好的命令。单行next命令改变了这个流程。例如有5行数据文本. 其中穿插着两个空行. 使用sed 's/^$/d' data1.txt

gawk

定义变量来保存数据, 使用算术和字符串操作符来处理数据. 使用结构化变成概念来微数据处理增加处理逻辑. 通过提取数据文件中的数据元素. 将其重新排列或格式化 生成格式化报告.

-F fs 字段分隔符 -f file 从指定文件中读取 -v var=value 定义gawk程序中的一个变量及其默认值 -mf N 处理的数据文件中的最大字段数 -mr N 数据稳重的最大数据行数 -W keyword 兼容模式和警告等级.

使用数据字段变量

  • $0 代表整个文本行
  • $1 文本行中的第一个数据字段
  • $2 文本行中的第二个数据字段
