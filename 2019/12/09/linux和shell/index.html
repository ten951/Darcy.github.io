shell命令

基本的bash shell命令

遍历文件

cd 路径名 用以指定想切换到的目录名. 如果没有指定目标路径名. 则切换到当期用户的主目录
pwd 可以显示出shell会话的当前目录，这个目录被称为当前工作目录。

文件和目录列表

ls -alf 显示当前目录下的文件和目录 默认是按照字母排序的 -F 可以区分文件和目录 -A 显示隐藏文件 -l 长列表格式输出

可以依据名称过滤. ?匹配一个字符 *多个字符

ls -l scri?t

处理文件

  1. 创建文件 touch
  2. 复制文件 cp source destination 将source复制到destination. -i选项可以提示是否要覆盖 -r会递归的将目录下的子文件复制过去.
  3. 链接文件 ln 分符号链接 -s 硬链接
  4. 重命名文件 mv 可以将文件或者目录移动到另一个位置或重新命名 移动文件不会影响文件的inode和时间戳 -i可以提示是否要覆盖
  5. 删除文件 rm -i 提示 -f 强制删除 -rf是非常暴力的
  6. 创建目录 mkdir -p 可以创建多级目录
  7. 删除目录 rmdir 只能删除空目录. 结合rm命令使用
  8. 查看文件类型 file
  9. 查看整个文件 cat -n是给所有行添加行号. -b 只会给文本添加行号 more 分页查看整个文件 less是more的升级版本
  10. 查看部分文件 tail 从尾部查看文件. 默认10行. -n可以指定行数 -f 它允许你在其他进程使用该文件时查看文件 head 从头部查看文件

检测程序

ps 命令 -e 显示所有运行在系统上的进程 -f 显示拓展输出 -l 更多信息长格式输出.它只能显示某个特定时间点的信息。如果想观察那些频繁换进换出的内存的进程趋势，用ps命令就不方便了。

top 命令 实时检测进程.

第一行top 系统当期实际. 系统运行了多长时间. 登录的用户数. 及其系统的平均负载 最近1分钟 5分钟 最近15分钟
第二行task 显式多少进程处于运行, 休眠 停止或是僵化状态(僵化状态是指 子进程运行完成, 父进程还没有响应)
第三行cpu 显式了进程所属的宿主 是用户us还是系统sy和进程的状态.运行 空间和等待. 将CPU分成几类输出.

us表示用户态
ni表示低优先级用户态
sy 内核态
id 空间状态 不包含等待IO的时间
wa 等待IO的时间
hi 处理硬中断的时间
si 处理软中断的时间
st 在虚拟机中的时间. 被其他虚拟机占用的时间
guest 虚拟化运行其他操作系统的时间 运行虚拟机的cpu时间
gnice 低优先级虚拟机的时间

cpu使用率的计算 = 1 - 空间时间/总CPU时间

实际上为了计算CPU使用率. 性能工具一般会取间隔一段时间例如3秒的两次值, 做差后, 在计算出这段时间内的平均CPU使用率.

平均CPU使用率 = 1 - (空间时间new-空间时间old)/(总cpu时间new-总cpu时间old);

结束进程

linux进程信号:

  1. HUP 挂起
  2. INT 中断
  3. QUIT 结束运行
  4. KILL 无条件终止
  5. SEGC 段错误
  6. TERM 尽可能终止
  7. STOP 无条件停止运行，但不终止
  8. TSTP 停止或暂停，但继续在后台运行
  9. CONT 在STOP或TSTP之后恢复执行
  • kill命令. 必须是root用户. kill 3306 本质是发送term信号. 有时候会不管用. kill -s HUP 3306 在命令后面添加参数-s. 并添加信号名称. 能不能执行成功, 通过ps或者top命令查看.
  • killall命令 是强大的进程结束命令. killall http* 结束了所有http开头的进程

检测磁盘空间

linux文件系统将所有磁盘都并入一个虚拟目录中. 在使用新的存储媒体之前. 需要把它放到虚拟目录下. 这项工作称之为挂载.

  • mount 会输出当前系统挂载的设备列表. 输出分为4部分 on之前: 表示媒体设备的文件名. on之后到type之间: 媒体挂载到虚拟目录的挂载点, type之后: 文件系统的类型. ()内 已挂载媒体的访问状态 mount type device directory type代表文件类型. 后面的是文件目录

  • umount 卸载设备

  • df 某个设备上还有多少空间 -h 按照用户易读的方式显示.

  • du 命令显示当前目录下 所有的文件的磁盘使用情况 -c：显示所有已列出文件总的大小。-h：按用户易读的格式输出大小，即用K替代千字节，用M替代兆字节，用G替代吉字节。-s：显示每个输出参数的总计。

  • sort 排序数据 -n 识别数字进行排序, -t 指定文件分隔符 -k 指定按照pos位置排序. 例如 sort -t ':' -k 3 -n /etc/passwd 对/etc/passwd文件排序. 以:为分隔符形成数组(下标从1开始) 3位置的字符识别为数字, 进行排序. -r 降序

  • grep 搜索数据

理解Shell

shell不单单是一种CLI。它是一个时刻都在运行的复杂交互式程序。输入命令并利用shell来运行脚本会出现一些既有趣又令人困惑的问题。搞清楚shell进程以及它与系统之间的关系能够帮助你解决这些难题，或是完全避开它们。

类型

在/etc/passwd文件中记录了用户id和使用的shell信息

第7pos 表示shell的类型. 有很多种. bash, dash,zsh等

shell的父子关系

用于登录某个虚拟控制器终端或在GUI中运行终端仿真器时所启动的默认的交互shell，是一个父shell。

当运行bash命令时. 会生成一个子shell, 子shell也拥有CLI符号, 等待输入. 当输入命令 ps -f时 会看见三个进程

-bash 是爷爷进程 bash是父亲进程 ps -f是儿子进程. 观察PPID和PID的关系. 就明白了 通过ps -forest 命令可以更加直观的看到关系

进程列表

进程列表是另外一种创建子shell的方法. 将一组命令通过;的形式分割. 用()包起来’’(pwd;ls;cd/etc;pwd;cd;pwd;ls;echo $BASH_SUBSHELL)’’

echo $BASH_SUBSHELL 如果输入为0. 说明是父进程. 输出为1 或者更大, 说明是子进程. 在shell脚本中，经常使用子shell进行多进程处理。但是采用子在shell脚本中，经常使用子shell进行多进程处理。但是采用子

  1. 另外一种高效的子shell用法就是 后台模式 比如 sleep命令. 当输入sleep 10的时候. 会暂停10秒钟. 10s之后. shell窗口才可用, 当输入 sleep 10&时. 并不会阻塞当期shell窗口. 效果如下:

第一条[1] 表示后台作业任务号1. 第二条是后台任务的进程ID. 通过ps -f或者jobs -l 都可以看到任务的更多信息.

  1. 将进程列表置入后台 ‘’(sleep2;echo $BASH_SUBSHELL;sleep2)&’’ 即可以在子进程中完成繁重的工作, 又不会子shell的i/o受制于终端.
  2. 协程可以同时做两件事。它在后台生成一个子shell，并在这个子shell中执行命令。要进行协程处理，得使用coproc命令，coproc sleep 10 或者 coproc my_sleep { sleep 10; } 两种形式. 输入内容都是一致的. 作业号和作业进程ID.

什么是平均负载

通过直径top或者uptime命令. 可以了解系统的负载情况. 最后的三个数组依次是 1,5,15分钟的平均负载.

平均负载和CPU使用率是不一样的. 平均负载是指单位时间内, 系统处于可运行状态和不可中断状态的平均进程数. 也就是平均活跃进程数.

可运行状态时值正在使用CPU或者正在等待CPU的进程, 也就是常用ps命令看到的处于R状态的进程.

不可中断状态的进程则是正在处于内核态关键流程中的进程. 并且这些流程是不可打断的. 比如最常见的是等待硬件设备的IO设备. 也就是PS命令看到的D状态的进程. 例如一个进程向磁盘读写数据.为了保证数据一致性, 在得到回复前, 它是不能被其他进程或者中断锻打的. 这个时候的进程就处于不可中断的.不可中断状态实际上是系统对进程和硬件设备的一种保护机制.

我们还是要回到平均负载的含义上来，平均负载是指单位时间内，处于可运行状态和不可中断状态的进程数。所以，它不仅包括了正在使用 CPU 的进程，还包括等待 CPU 和等待 I/O 的进程。

而 CPU 使用率，是单位时间内 CPU 繁忙情况的统计，跟平均负载并不一定完全对应。

比如：CPU 密集型进程，使用大量 CPU 会导致平均负载升高，此时这两者是一致的；

I/O 密集型进程，等待 I/O 也会导致平均负载升高，但 CPU 使用率不一定很高；

大量等待 CPU 的进程调度也会导致平均负载升高，此时的 CPU 使用率也会比较高。

CPU上下文切换是什么意思

linux是一个多任务操作系统. 支持大于cpu数量的任务同时运行. 当然，这些任务实际上并不是真的在同时运行，而是因为系统在很短的时间内，将 CPU 轮流分配给它们，造成多任务同时运行的错觉。

在每个任务运行前. CPU都需要知道任务从哪里加载. 又从哪里开始运行.也就是说. 需要系统事先设置好CPU寄存器和程序计数器.

CPU上下文指的就是 : CPU 寄存器，是 CPU 内置的容量小、但速度极快的内存。而程序计数器，则是用来存储 CPU 正在执行的指令位置、或者即将执行的下一条指令位置。它们都是 CPU 在运行任何任务前，必须的依赖环境.
