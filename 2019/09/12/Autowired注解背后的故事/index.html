@Autowired注解背后的故事

发表于 2019-09-12 | 分类于 Spring
字数统计: 2.3k 字 | 阅读时长 ≈ 11 分钟

spring注入方式

0 AUTOWIRE_NO

1 AUTOWIRE_BY_NAME AutowiredMethodElement

通过名字注入. 规则是set开头. 后面的要和bean的名字相同. 例如setOrderService, 并且OrderService必须是被spring ioc容器托管的.

method.invoke(bean, arguments);

2 AUTOWIRE_BY_TYPE AutowiredFieldElement

通过类型注入, field.set(bean, value);

3 AUTOWIRE_CONSTRUCTOR 构造函数注入

通过构造函数注入. 但是会选择合适的构造函数的. 这个的执行是在new java对象的时候执行的并且bd的注入模型必须是3

AutowiredAnnotationBeanPostProcessor类注册和执行

AutowiredAnnotationBeanPostProcessor是BeanPostProcessor的子类. so它的执行就是是在AbstractApplicationContext#refresh():

1.registerBeanPostProcessors(beanFactory)-> PostProcessorRegistrationDelegate.registerBeanPostProcessors(beanFactory, this) 将BeanPostProcessors类的注入初始化了

BeanPostProcessor pp = beanFactory.getBean(ppName, BeanPostProcessor.class)一直跟踪这个方法,路径是
AbstractBeanFactory#getBean -> AbstractBeanFactory#doGetBean() -> AbstractAutowireCapableBeanFactory#createBean() -> AbstractAutowireCapableBeanFactory#doCreateBean() 也就是下面的代码

先了解一下InstantiationAwareBeanPostProcessor接口 这个接口主要作用在于目标对象的实例化过程中需要处理的事情，包括实例化对象的前后过程以及实例的属性设置

case 1 AutowiredAnnotationBeanPostProcessor#postProcessMergedBeanDefinition方法

这个方法的主要作用就是将那些标记@Autowired、@Value、@Inject注解的Element封装成InjectionMetadata. 放入injectionMetadataCache中.

case 2 postProcessProperties 执行属性注入

@Autowired和@Resource这两个注解是我们开发过程中经常使用的注解
@Autowired默认按类型装配，默认情况下必须要求依赖对象必须存在，如果要允许null值，可以设置它的required属性为false，例如：@Autowired(required=false) ，如果我们想使用名称装配可以结合@Qualifier注解进行使用
@Resource，默认安装名称进行装配，名称可以通过name属性进行指定，如果没有指定name属性，当注解写在字段上时，默认取字段名进行安装名称查找，如果注解写在setter方法上默认取属性名进行装配。当找不到与名称匹配的bean时才按照类型进行装配。但是需要注意的是，如果name属性一旦指定，就只会按照名称进行装配。
@Autowired、@Value、@Inject 注解的实现大体逻辑是先获取哪些需要注入的属性，然后调用反射进行赋值

finishBeanFactoryInitialization()->DefaultListableBeanFactory#preInstantiateSingletons()

解决注册表中的也就是@AutoConfigurationPackage注解扫描注册DefaultListableBeanFactory#beanDefinitionNames属性中的注入 最终的调用路径也是AbstractBeanFactory#getBean -> AbstractBeanFactory#doGetBean() -> AbstractAutowireCapableBeanFactory#createBean() -> AbstractAutowireCapableBeanFactory#doCreateBean().
